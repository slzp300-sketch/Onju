# 온주 출시 로드맵

목표: **플레이스토어 + 앱스토어 배포**, 카카오/구글 로그인 완전 동작
(2026-06-11 기준 — Supabase 백엔드 마이그레이션 완료 시점)

## ✅ 완료된 것

- Supabase 백엔드 (인증 + 전체 데이터 동기화 + 소모임 다중 사용자)
- 이메일/구글 로그인 — 웹·안드로이드 모두 검증 완료
- 웹 프로덕션 배포 (onju-seven.vercel.app)
- 안드로이드 디버그 APK 에뮬레이터 검증

## 🔄 1. 카카오 로그인 마무리 (진행 중)

- [ ] 카카오 "추가 기능 신청"(이메일 동의항목) 승인 대기
- [ ] 승인 후: 카카오계정(이메일) → 필수 동의 설정
- [ ] 에뮬레이터 + 웹에서 카카오 로그인 테스트 (코드 수정 불필요)
- [ ] 릴리스 키스토어의 키 해시를 카카오 콘솔 Android 플랫폼에 추가 등록

## 📋 2. 스토어 공통 필수 준비물

- [ ] **개인정보처리방침 페이지** (URL 필수 — 양 스토어 모두 요구, 카카오 비즈앱 심사에도 필요)
- [ ] **서비스 이용약관 페이지**
- [ ] **회원 탈퇴(계정 삭제) 기능** — 구글·애플 정책상 계정 생성이 있으면 앱 내 삭제 기능 의무
- [ ] 로그인 화면 레거시 "직" 로고 → 온주 브랜딩 교체
- [ ] 스토어 등록 자료: 스크린샷(폰 기준 최소 2장), 앱 설명(짧은/긴), 512px 아이콘, 그래픽 이미지

## 🤖 3. 플레이스토어 (안드로이드)

- [ ] Google Play Console 개발자 계정 등록 ($25 일회성)
- [ ] **릴리스 SHA-1**을 구글 클라우드 Android OAuth 클라이언트에 추가
      (디버그 SHA-1만 등록돼 있으면 릴리스 빌드에서 구글 로그인 실패)
- [ ] `gradlew bundleRelease`로 서명된 AAB 빌드
- [ ] 데이터 보안 양식 작성 (수집 데이터: 이메일, 이름, 사용자 생성 콘텐츠)
- [ ] 내부 테스트 트랙 업로드 → 실기기 검증 → 프로덕션 심사 제출

## 🍎 4. 앱스토어 (iOS) — 가장 큰 신규 작업

- [ ] **Mac 확보** (Xcode 없이 iOS 빌드 불가 — 실물 Mac, 클라우드 Mac 대여, 또는 CI 서비스)
- [ ] Apple Developer Program 가입 ($99/년)
- [ ] `npx cap add ios` + 아이콘/스플래시 생성
- [ ] **Sign in with Apple 구현 의무** — 소셜 로그인(구글/카카오)을 제공하는 앱은
      애플 심사 지침상 Apple 로그인도 필수. Supabase Apple 제공자 + capgo 플러그인으로 구현
- [ ] 구글 iOS OAuth 클라이언트 생성, 카카오 iOS 플랫폼 등록
- [ ] TestFlight 검증 → 심사 제출

## 🔧 5. 출시 전 기술 정비 (우선순위순)

- [ ] **Supabase 요금제**: 무료 티어는 1주 미사용 시 일시정지 — 출시 전 Pro 전환 검토
- [ ] **알림 동작 확인**: 현재 웹 Notification API 기반 — 네이티브 WebView에서 동작 여부 확인,
      안 되면 @capacitor/local-notifications로 교체
- [ ] 오프라인 쓰기 유실 대응 (재시도 큐) — 선택, 출시 후도 가능
- [ ] 테스트 계정(onju.tester1/2) Supabase에서 삭제
- [ ] 에러 모니터링(Sentry 등) 도입 검토 — 선택

## 권장 진행 순서

```
기능 개발 계속 (지금)
  └→ 카카오 승인 나는 대로 로그인 테스트 (병행)
  └→ 2번 공통 준비물 (개인정보처리방침·계정삭제는 개발 작업이므로 미리)
       └→ 3번 플레이스토어 (안드로이드 먼저 출시)
            └→ 4번 iOS (Mac 확보 후)
```

앱스토어는 Mac과 Apple 로그인 구현이 필요해서, **플레이스토어 먼저 출시하고 iOS는 2차로** 가는 걸 권장.
