## 1. 앱 UX · 화면 설계

### 온보딩 플로우 (`/onboarding`)

4단계 스텝 위저드. 각 스텝은 풀스크린, 상단 프로그레스 바(25% → 50% → 75% → 100%).

```
Step 1 — 월간 목표 입력
- 텍스트 인풋 1개 (placeholder: "예) 직장에서 선한 영향력 실천하기")
- "지금 건너뛰기" 링크 → 빈 목표로 다음 진행
- CTA: "다음 →"

Step 2 — 개인 루틴 설정 (Track A)
- 루틴 이름 입력 + 빈도 선택 (매일 / 평일 / 주말)
- "루틴 추가" 버튼으로 복수 추가 가능
- 최소 1개 이상 추가해야 다음 활성화

Step 3 — 신앙 루틴 선택 (Track B)
- 기도 · 말씀 기본 선택된 상태로 시작
- 추가 템플릿 그리드 표시 (4개씩 2열)
- 탭 없이 바로 토글

Step 4 — 소모임 탐색
- 공개 그룹 3개 미리보기 카드
- "소모임 둘러보기" / "나중에 할게요" 선택
```

**컴포넌트**: `OnboardingLayout.tsx` → step state 관리, Framer Motion `AnimatePresence` 로 스텝 전환 슬라이드

### 축하 모달 컴포넌트 (`SlotUnlockModal.tsx`)

```tsx
// 애니메이션 시퀀스:
// 1. 모달 카드 scale(0.8 → 1) + opacity(0 → 1), 300ms
// 2. 슬롯 아이콘 stagger 등장 (100ms 간격)
// 3. canvas confetti 실행 (보라 #7F77DD, 노랑 #EF9F27)
// 4. 햅틱: navigator.vibrate([50, 30, 50, 30, 100])

const SlotUnlockModal = ({ newCount }: { newCount: number }) => {
  useEffect(() => {
    confetti({ colors: ['#7F77DD', '#EF9F27', '#1D9E75'], particleCount: 100 });
    if ('vibrate' in navigator) navigator.vibrate([50, 30, 50, 30, 100]);
  }, []);

  return (
    // Framer Motion AnimatePresence 안에 위치
    <motion.div initial={{ scale: 0.8, opacity: 0 }} animate={{ scale: 1, opacity: 1 }}>
      <h2>슬롯 +1 해금!</h2>
      <p>지난 주 달성률이 80%를 넘었어요.</p>
      <SlotGrid total={newCount} animated />
    </motion.div>
  );
};
```

```typescript
// 매일 2개 랜덤 선택 (date seed 활용 → 같은 날 앱 재진입 시 동일 질문 유지)
const getDailyQuestions = (date: string): [string, string] => {
  const seed = parseInt(date.replace(/-/g, ''));
  const idx1 = seed % identityQuestions.length;
  const idx2 = (seed + 3) % identityQuestions.length;
  return [identityQuestions[idx1], identityQuestions[idx2]];
};

// 답변: '잘함' | '보통' | '아쉬움' + 선택적 텍스트 메모
```

```tsx
  if (rate === 0)   return 'var(--color-border-tertiary)'; // 미기록
```
